# Reopen Gate — Puzzle #71 Research

Research stays **closed** unless **one** of the following primary-evidence gates is met.

Forum guesses, prefix lists, numerology, and generic wallet demos **do not** open the gate.

---

## Gates (any one sufficient to reopen *narrow* work)

| ID | Gate | Opens |
|----|------|--------|
| **G1** | Creator-linked **wallet backup** (verifiable provenance) | Narrow verify / constrained derive |
| **G2** | Verifiable **MPK / root / xpub / chain code** tied to Puzzle | Family + sequence check vs known `A[i]` |
| **G3** | Exact historical **mask `T` source code** (dated / attributable) | Fix `T`; still need `K` source for full solve |
| **G4** | **Second authenticated creator message** containing **technical material** (not a copy of 2017-04-27 wording alone) | New archive + possible `T`/tooling |
| **G5** | Pre-existing artifact tying a **generator + seed material** to Puzzle addresses | Narrow candidate set |

---

## Explicit non-gates

- “Could be Electrum / Armory / Casascius / BIP32”  
- Cascade / random / timestamp / passphrase lists without G1–G5  
- Vanity / prefix / numerology  
- Explorer mirrors of known TXIDs/addresses  
- Bitaddress / stock paper-wallet HTML  
- Re-running Monte Carlo without new evidence  
- Copies of the known 2017 `saatoshi_rising` post

---

## Process when a gate trips

1. Document artifact in `LINKAGE_EVIDENCE_MATRIX.tsv` with Grade **A** or strong **B**.  
2. Record `verified_original_date`, SHA-256, archive URL.  
3. Update `RESEARCH_CLOSURE.md` status only after human review.  
4. Open the **smallest** test that the artifact justifies — never a full keyspace grind.
